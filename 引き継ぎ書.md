## 5. 生徒画面の構造（英語）
`英語 → 学年(中1/中2/中3) → Unit を選ぶ → 10問`
- index.html の `SUBJECTS`（410行あたり）の english エントリ：
  - `gradeFirst:true, grades:true`
  - `units` は**学年ごとの連想配列**：`{ '中1':[...14...], '中2':[...20...], '中3':[] }`
  - 中3は空配列なので、いま中3を選ぶと「準備中」表示になる（`showUnits`が空を検出）
- Unit選択画面には各Unitに **「正解数/総単語数」バッジ**が出る（分子=last_result=true の語数、分母=そのUnitの総語数、全問正解は緑）。進捗は本人分のみ読み込み（RLS準拠）
- ナビ関数: `renderMenu → showGradeList(学年) → showUnits(s,grade) → startQuiz(slug,unit,grade)`。社会は別ルート（showTopics/showGrades）で共存

## 6. データ構造（Supabase・要点）
- questions(id, subject_id, unit, grade, format, prompt, answer, choices, explanation, is_active)
  - 英単語は format='vocab', prompt=英語, answer=日本語, choices=null（誤答は同じ unit+grade の他の答えから自動生成）, unit='Unit1'等, grade='中1'等
- student_question_progress(student_id, question_id, correct_count, wrong_count, last_result(boolean), last_seen_date)
- daily_sessions … 各セット実施ログ（再挑戦も毎回1行。旧uniqueは撤廃）。topic="Unit/学年"
- students … total_points に加え **xp（累積・減らない=レベル）/ coin（残高=景品用）/ last_login_bonus_date** を保持。total_points は xp と同期（admin互換）
- 新テーブル: daily_unit_perfect（全問正解+5を同日同Unit1回に制御）/ unit_mastery（Unit制覇+5を一生1回）/ point_ledger（XP・コイン入出金台帳＝将来のコイン消費/引き換え待ちリストの土台。本人＋先生が閲覧可）
- RLS有効。生徒は自分の行だけ読める。加点・採点はサーバー側RPC（改ざん不可）
- 主要RPC: get_daily_questions(slug,limit,unit,grade) / submit_daily_quiz(slug,answers,unit,grade) / **claim_login_bonus()** / calc_level(points)=**50ptで1Lv**

### ポイント制度（2026-07-01 改訂・migrate_points_v2.sql 適用済み）★最新
- **2通貨**: XP(累積・レベル)＝コイン(残高・景品用) を**同額・同時**付与
- 貯め方: ログイン+1(1日1回) / 参加賞=1セット実施+1(1日1回) / 10問全問正解+5(同Unit1日1回) / Unit制覇=全単語を1度以上正解+5(一生1回)
- ストリーク: 3日+5 / 7日+15 / 14日+30 / 30日+50(以降30日ごと繰返)。判定=その日に参加賞が入った(1セット実施)こと。日本時間。連続と通算(total_days)の2本立て
- レベル: 50ptごと(Lv=xp/50+1)。UIは「次まであと◯pt」表示
- 景品交換UIは**準備中(グレーアウト)**。コイン消費/引き換えは景品カタログ確定後に解禁(保管仕様: 選択式ボタンでcoinだけ減らしpoint_ledgerにkind=spendで履歴、XP不変)
- index.html: ログイン時 claim_login_bonus 呼出／結果画面にXP+コイン+内訳(参加賞/連続/全問正解/制覇)／マイページにコイン残高・景品準備中・「どうすればポイントが入る?」ガイド画面(s-guide)

## 7. 写真→CSVの変換ルール（中1・中2で確定した方針）
- 出力は `単語,意味` の1行1語CSV（admin.htmlのCSV一括にも貼れる形式）
- **全部入れる**：短縮形・固有名詞・重要表現・フレーズも含める
- **Unitごとに1ファイル**にセクション分け（Unit1〜、言語活動N、LetsReadMore など）
- 意味の区切りは日本語読点「、」を使う（ASCIIカンマは使わない）
- ⚠️**英語側にASCIIカンマを入れない**：CSV取り込みは最初のカンマまでを英単語とみなすため。
  例: `Best wishes,`→`Best wishes`、`Oh, no.`→`Oh no.`、`Go, ~!`→`Go ~!`、`~, right?`→`~ right?`、`Yes, please.`→`Yes please.` のようにカンマを除去
- ファイル名例: `Unit1_HajinsDiary.csv` / `言語活動1.csv` / `LetsReadMore.csv`
- 横向き・低解像度写真はOCR精度が落ちる。作った後に「怪しい所リスト」を出して本と照合してもらうこと（中2のLetsReadMore/言語活動11は最初ミスして再送写真で修正した実績あり）
